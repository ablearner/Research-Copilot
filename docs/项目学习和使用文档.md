这份文档面向两类场景：

- 想快速知道当前仓库该从哪里看起

如果你只想跑项目，先看 [系统运行指南.md](系统运行指南.md)。
如果你想讲项目或准备面试，先看 [系统学习与面试指南.md](系统学习与面试指南.md)。

## 1. 先用一句话理解项目

`Research-Copilot` 是一个多智能体科研助手。

它的上层负责研究任务编排、候选论文筛选、导入和研究集合问答；下层负责文档解析、图表理解、混合检索和 grounded answer。

按代码分层可以直接记成四层：

- `app / components / lib`
  前端研究工作区、状态控制和后端代理
- `apps/api`
  FastAPI 入口、路由、依赖注入和 runtime 装配（含 `research_runtime.py`）
- `services/research + agents + skills/research`
  research 业务编排、specialist agents 和 16+ 研究技能类
- `rag_runtime + tools + retrieval + adapters + memory + reasoning + tooling + mcp`
  底层执行、检索、记忆系统、推理策略、工具框架和 MCP 协议

## 3. 目录速查

### `apps/api/`

FastAPI 应用、依赖注入和 HTTP 路由。

重点文件：

- `main.py`
- `runtime.py`
- `research_runtime.py`
- `routers/research.py`

### `services/research/`

research 主业务、导入、报告、workspace 持久化和 Supervisor Graph orchestration。

### `agents/`

高层决策和研究任务专用 agent。

当前最值得记住的分工：

- `ResearchSupervisorAgent`
  决定下一步做什么
- `LiteratureScoutAgent`
  负责 search / discovery
- `ResearchKnowledgeAgent`
  负责 evidence gathering
- `ResearchWriterAgent`
  负责把已有证据组织成回答或报告
- `PaperAnalysisAgent`
  负责对比、推荐和 explanation
- `ChartAnalysisAgent`
  负责 figure 列表、visual anchor 和单图分析
- `PreferenceMemoryAgent`
  负责用户偏好学习与记忆，驱动 `UserProfileMemory` 更新
- `GeneralAnswerAgent`
  负责通用问答（不依赖 research context 的简单问题）

### `rag_runtime/`

底层 RAG runtime、tool-first pipeline 和运行时状态。

### `memory/`

research 侧工作记忆、会话记忆、长期记忆、用户画像记忆和论文知识记忆。

### `retrieval/`

检索、重排和 evidence bundle 构造。

### `tools/`

文档、图表、检索、回答等稳定工具层。

### `sdk/`

本地 SDK 封装和 CLI runtime profile。

### `components/`、`app/`、`lib/`

当前前端工作区与页面入口。

## 4. 你最该记住的五个事实

1. 当前主产品不是单文档问答台，而是科研助手。
2. `/research/agent` 是对外统一主入口。
3. `RagRuntime` 不是顶层大脑，而是执行层。
4. 项目里同时存在 research 侧 memory 和 RAG runtime 侧 session memory，两者不要混为一谈。
5. 默认配置下，向量库是 `milvus`，图存储是 `memory`，session memory 是 `auto`，long-term memory 是 `json`。
6. `skills/research/` 下的 16+ 研究技能类是业务逻辑的核心组件，被 agents 和 services 广泛调用。
7. MCP 协议用于对外接入外部工具（Zotero）和对外暴露 RAG 能力。

## 5. 当前 research 接口脑图

至少要知道这些接口：

- 会话
  - `GET/POST /research/conversations`
  - `GET/PATCH/DELETE /research/conversations/{conversation_id}`
- 主入口
  - `POST /research/agent`
- 任务
  - `POST /research/tasks`
  - `GET /research/tasks/{task_id}`
  - `POST /research/tasks/{task_id}/run`
  - `GET /research/tasks/{task_id}/report`
- 检索与导入
  - `POST /research/papers/search`
  - `POST /research/tasks/{task_id}/papers/import`
  - `POST /research/tasks/{task_id}/papers/import/jobs`
  - `GET /research/jobs/{job_id}`
- figure 与 TODO
  - `GET /research/tasks/{task_id}/papers/{paper_id}/figures`
  - `POST /research/tasks/{task_id}/papers/{paper_id}/figures/analyze`
  - `PATCH /research/tasks/{task_id}/todos/{todo_id}`
