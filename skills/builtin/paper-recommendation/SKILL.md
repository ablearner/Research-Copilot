# Paper Recommendation Skill

## Agent 路由策略
Supervisor 应调度：
1. **PreferenceMemoryAgent** — 基于用户兴趣档案生成个性化推荐

## 使用场景
当用户想要基于个人兴趣获得论文推荐时激活此技能。典型请求：
- "推荐几篇最近值得看的论文"
- "根据我的兴趣，有什么新论文"
- "帮我挑几篇 CV 方向的好论文"

## 执行步骤

1. **兴趣理解**：
   - 从用户长期偏好记忆中提取研究兴趣主题
   - 结合当前请求理解具体推荐意图
   - 如果用户指定了主题，优先按指定主题推荐
   - 如果用户没指定（泛推荐），从兴趣档案中选取 top 主题
2. **论文搜索**：
   - 在多个学术数据源中搜索匹配论文
   - 默认搜索近 30 天的新论文
3. **个性化排序**：
   - 结合用户兴趣权重、论文时效性、来源质量综合排序
   - 每篇论文生成个性化推荐理由（解释为什么推荐给这位用户）
4. **分组呈现**：按主题分组展示推荐结果

## 质量标准
- 推荐理由必须个性化：引用用户的具体兴趣主题解释相关性
- 不接受模板化理由（如"你可能感兴趣"）
- 默认推荐 4-6 篇，每个兴趣主题至少 1-2 篇
- 推荐结果应该有多样性，不要全是同一子方向

## 输出格式
- 按主题分组的论文列表
- 每篇：标题、来源、年份、个性化推荐理由、链接
- 可选：附带用户兴趣档案摘要

## 约束
- 推荐基于实际搜索结果，不虚构论文
- 如果用户兴趣档案为空（新用户），提示用户先进行几次研究对话以建立兴趣档案
- 跟随用户语言
